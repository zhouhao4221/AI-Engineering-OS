# 03 · 构建

AI 辅助开发实践。团队如何与 AI 一起编写、审查和交付代码。

---

## 目的

本章节定义日常开发实践，使 AI 成为可靠的贡献者，而非产出不可预测结果的来源。

---

## 待覆盖的主题

- **开发循环** — 标准流程：需求 → 提示 → 生成 → 审查 → 精炼 → 提交
- **PR 标准** — 每个 PR 必须包含 AI 解释、风险分析和测试建议（见工程规则）
- **AI 辅助代码审查** — 如何在审查中使用 AI 而不绕过人工判断
- **测试生成** — 生成真正有用的 AI 测试的模式
- **重构模式** — 如何安全地使用 AI 进行大规模重构
- **上下文管理** — 如何为编码任务给 AI 提供正确的上下文（包含什么、排除什么）

---

## PR 检查清单

本系统中的每个 PR 必须包含：

- [ ] AI 解释：AI 生成了什么，人类做了哪些修改
- [ ] 风险分析：可能出什么问题、边界情况、受影响的依赖
- [ ] 测试建议：哪些测试覆盖了本次变更，还有哪些空白
