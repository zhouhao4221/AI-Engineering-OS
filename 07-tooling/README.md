# 07 · 工具

AI 工具与自动化技术栈。AI 驱动工程体系的基础设施层。

---

## 目的

工具是原则变为实践的方式。本章节记录正在使用的工具、其集成方式和维护方法。

---

## 当前技术栈

| 类别 | 工具 |
|------|------|
| AI 编码 | Claude Code、Codex |
| 版本控制 | Git + 结构化 PR 模板 |
| CI/CD | 按项目配置 |
| 提示管理 | 本仓库（04-prompt） |

---

## 待覆盖的主题

- **工具选型标准** — 如何评估和采用新的 AI 工具
- **Claude Code 配置** — CLAUDE.md 规范、钩子、权限设置
- **CI/CD 集成** — 流水线中的 AI 审查步骤、自动化提示测试
- **提示版本管理** — 用于追踪提示变更和输出质量的工具
- **本地开发环境** — AI 辅助开发的标准环境配置
- **监控** — 随时间追踪 AI 贡献比率和输出质量

---

## 工具采用流程

1. 对照四大核心原则进行评估
2. 记录使用场景和预期输出
3. 以明确的成功指标进行限时试用
4. 若采用：在本章节记录，并在 `04-prompt` 中创建相关提示
5. 若拒绝：记录原因，避免后续重复评估同一工具
