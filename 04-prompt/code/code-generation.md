# 代码生成提示

## 目的

根据接口定义和约束生成实现代码。适用于功能函数、模块、服务类等。不适用于核心业务规则（应人工编写）。

## 输入

- `{{接口定义}}`：函数签名、类型定义、API 契约
- `{{实现示例}}`：代码库中 1-2 个类似功能的现有实现
- `{{约束}}`：技术栈、禁用的库、不能修改的接口
- `{{验收标准}}`：行为描述 + 边界条件

## 提示

```
你是一名熟悉此代码库风格的工程师。请根据以下信息实现代码。

接口定义：
{{接口定义}}

代码库中的类似实现（供参考风格，不要直接复制）：
{{实现示例}}

约束：
{{约束}}

验收标准：
{{验收标准}}

要求：
- 严格遵守已有接口，不要引入未定义的新类型
- 错误处理：所有外部依赖调用须处理失败情况
- 不要添加超出验收标准要求的功能
- 如果某个约束互相冲突，先指出再实现

只输出代码，不要解释每行的含义。代码之后简述你做了哪些关键决策。
```

## 预期输出

- 符合接口定义的完整实现代码
- 3-5 句关键决策说明（如：为何选择某种错误处理方式）
- 需要人工确认的假设列表（若有）

## 备注

- 版本：v1.1
- 优化历史：v1.0 → v1.1：增加"不要添加额外功能"约束，减少过度实现问题
- 已知限制：对依赖大量隐式状态的遗留代码效果差；使用前先将相关全局状态显式化
