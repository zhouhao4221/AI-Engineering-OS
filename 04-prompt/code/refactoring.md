# 代码重构提示

## 目的

在不改变外部行为的前提下改善代码结构，提升 AI 可操作性（可读性、边界清晰度、职责单一性）。

## 输入

- `{{目标代码}}`：需要重构的代码片段
- `{{重构目标}}`：具体要改善什么（例如：拆分函数、消除全局状态、提取接口）
- `{{不能改动}}`：对外接口、已有测试依赖的行为、第三方调用方

## 提示

```
请对以下代码进行重构。目标是改善结构，不改变外部行为。

原始代码：
{{目标代码}}

重构目标：
{{重构目标}}

不能改动的部分：
{{不能改动}}

要求：
- 保持所有对外接口签名不变
- 不引入新的运行时依赖
- 每个函数职责单一，名称反映其意图
- 输出重构后的完整代码

重构完成后，列出你做了哪些改动，以及每项改动为何有助于实现重构目标。
```

## 预期输出

- 重构后的完整代码（不是差异，是全文）
- 改动清单：每条一句话说明改了什么、为什么这样改
- 风险提示：有哪些行为变化需要人工核查（若有）

## 备注

- 版本：v1.0
- 优化历史：初始版本
- 已知限制：对混合了多个层职责的"大泥球"代码，重构前需先人工确定边界；直接交给 AI 会得到错误的拆分
