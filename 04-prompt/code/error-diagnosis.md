# 错误诊断提示

## 目的

分析运行时错误、测试失败或 AI 生成代码与预期不符的情况，找出根本原因并给出修复方向。适用于 AI 多轮生成后仍未达标、错误堆栈难以定位的场景。

## 输入

- `{{错误信息}}`：完整的错误信息或堆栈（stack trace）
- `{{相关代码}}`：出错的函数或模块代码
- `{{预期行为}}`：这段代码应该做什么
- `{{上下文}}`：调用方式、输入数据、环境信息（可选）

## 提示

```
你是一名有丰富调试经验的工程师。请分析以下错误并找出根本原因。

错误信息：
{{错误信息}}

相关代码：
{{相关代码}}

预期行为：
{{预期行为}}

上下文（调用方式/输入数据/环境）：
{{上下文}}

请按以下步骤输出：
1. **根本原因**：用一句话说明错误的根本原因（不是表象）
2. **原因分析**：详细解释为什么会出现这个错误（3-5 句）
3. **修复方向**：给出 1-3 个修复选项，说明各自的权衡
4. **推荐方案**：选择一个方案并说明原因
5. **验证方法**：如何确认修复是否有效

如果信息不足以定位根本原因，先列出你需要哪些额外信息，再给出最可能的假设。
```

## 预期输出

- 明确的根本原因（而非"可能是 X 问题"）
- 结构化的修复选项对比
- 验证方案，避免重复犯同类错误
