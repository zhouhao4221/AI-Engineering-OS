# PR 代码审查提示

## 目的

对 Pull Request 进行初轮 AI 审查，生成结构化审查意见，供人工审查者参考。不替代人工审查。

## 输入

- `{{代码差异}}`：PR 的完整 diff
- `{{PR 描述}}`：PR 标题 + 说明（含 AI 解释、风险分析字段）
- `{{相关接口}}`：被修改代码依赖或实现的接口定义
- `{{测试覆盖}}`：本次 PR 新增或修改的测试

## 提示

```
请审查以下 Pull Request，生成结构化审查意见。

PR 描述：
{{PR 描述}}

代码差异：
{{代码差异}}

相关接口定义：
{{相关接口}}

新增/修改的测试：
{{测试覆盖}}

审查维度（按重要性排序）：

1. 正确性
   - 实现是否与接口定义一致？
   - 是否存在明显的逻辑错误或边界条件遗漏？
   - 错误处理是否完整？

2. 安全性
   - 是否存在注入风险、权限绕过或敏感数据暴露？
   - 外部输入是否经过验证？

3. 可维护性
   - 函数职责是否单一？
   - 是否引入了难以理解的隐式依赖？

4. 测试充分性
   - 测试是否覆盖了主要边界情况？
   - 是否有明显的测试空白？

输出格式：
- 必须修改（阻塞合并）：[问题列表]
- 建议改进（不阻塞）：[建议列表]
- 值得关注（供人工审查者注意）：[观察列表]
- 整体评估：一句话总结

对每个问题，指明所在文件和行号（如可识别）。
```

## 预期输出

- 按三类分组的结构化审查意见
- 每条意见指向具体代码位置（文件 + 行号）
- 整体风险评估（低 / 中 / 高）

## 备注

- 版本：v1.2
- 优化历史：v1.0 → v1.1：增加安全性维度；v1.1 → v1.2：要求指明行号，减少模糊审查意见
- 已知限制：对业务逻辑正确性的判断能力有限，需提供充分的领域上下文；AI 审查擅长发现模式性问题，不擅长发现需要业务知识才能识别的逻辑错误
