# 需求结构化提示

## 目的

将模糊的需求描述转化为结构化输入，使其可以直接用于后续的 AI 辅助设计和开发。这是工作流的第一步 — 输入质量决定所有后续输出质量。

## 输入

- `{{原始需求}}`：来自产品、用户或会议记录的原始文字描述

## 提示

```
请将以下原始需求转化为结构化格式。

原始需求：
{{原始需求}}

请填写以下结构（每个字段必须明确回答，不能留空或写"待定"）：

目标：[一句话，这个需求期望达成什么用户或业务目标]
背景：[为什么现在要做这个，触发这个需求的场景是什么]
功能描述：[具体要做什么，尽量具体到交互或数据变化]
约束：[技术限制、兼容性要求、不能做的事]
验收标准：[3-5 条明确的完成判定条件，可测试的]
排除范围：[明确这次不做什么，避免范围蔓延]
依赖：[这个需求依赖哪些其他模块、服务或数据]

如果原始需求中有任何内容无法确定，在最后列出"待澄清问题"，而不是自行填写假设内容。
```

## 预期输出

- 填写完整的结构化需求文档
- 如有信息缺失，列出具体的待澄清问题（问题要可以直接问给产品或业务方）
- 不应输出解释或建议，只输出结构化结果

## 备注

- 版本：v1.1
- 优化历史：v1.0 → v1.1：增加"排除范围"字段，解决 AI 生成方案时频繁超出需求范围的问题
- 使用场景：需求评审前、开始设计前；不适合用于已经明确的技术任务（那类任务直接用代码生成提示）
