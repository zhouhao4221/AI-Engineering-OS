# 架构设计提示

## 目的

为新功能或系统生成架构方案初稿，供人工评估和决策。输出是选项而非决定 — 最终架构决策由人工做出。

## 输入

- `{{需求描述}}`：功能目标（用结构化模板填写）
- `{{现有系统}}`：相关的现有模块、数据流、技术栈
- `{{约束}}`：必须使用的技术、团队能力限制、性能/安全要求
- `{{规模预期}}`：数据量级、并发量、团队维护能力

## 提示

```
请为以下需求生成架构设计选项。

需求：
{{需求描述}}

现有系统背景：
{{现有系统}}

约束：
{{约束}}

规模预期：
{{规模预期}}

请提供 2-3 个架构选项，每个选项包含：
1. 一句话描述方案核心思路
2. 主要组件及其职责（用简单的文字图或列表）
3. 优势：解决了什么问题
4. 劣势：引入了什么复杂性或风险
5. 适用场景：这个方案在什么情况下是最优选择

最后推荐一个方案，并说明推荐理由。不要在没有更多信息的情况下直接给出唯一方案。
```

## 预期输出

- 2-3 个有实质差异的架构选项（不是同一方案的微调）
- 每个选项有清晰的权衡分析
- 一个推荐方案 + 推荐理由
- 做最终决策前需要澄清的问题（若有）

## 备注

- 版本：v1.0
- 优化历史：初始版本
- 已知限制：AI 对团队具体能力和历史包袱不了解；"现有系统"字段填写越详细，方案越贴合实际；避免填写过于抽象的需求描述
