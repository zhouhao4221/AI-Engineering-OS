# AI 设计提示模板

生成 UI 时注入给 AI 的上下文模板。使用前将占位符替换为项目实际内容，或直接引用 `ui-style-guide.md` 的对应章节。

---

## 基础上下文模板

每次让 AI 生成 UI 组件或页面时，在提示开头附加：

```
【项目设计约束】
品牌调性：[从 ui-style-guide.md 复制]
主色：[色值]
背景色 / 文字色：[色值]
字体：[字体栈]，正文 [size]，标题 [size]
间距基准：[如 4px]
圆角：[如 统一 4px]
阴影：[如 不使用，用边框代替]
图标库：[库名]

【禁止项】
[列出项目明确不允许的样式，如：不用渐变、不用大面积彩色]

【参考组件】
[可选：粘贴一段现有组件代码，让 AI 推断风格]
```

## 场景模板

### 新建组件

```
按照上方设计约束，生成一个 [组件名] 组件。

要求：
- 支持以下状态：默认 / 悬停 / 禁用 / 加载（按需选择）
- Props：[列出必要参数]
- 不引入新的颜色或间距值，只用约束中已定义的
```

### 页面布局

```
按照上方设计约束，生成 [页面名] 的布局结构。

页面包含：[模块列表，如：顶部导航、左侧菜单、内容区、底部操作栏]
交互说明：[如：菜单可折叠，内容区自适应剩余宽度]
不需要实现业务逻辑，只需结构和样式。
```

### 风格对齐检查

```
检查以下代码是否符合设计约束，列出不一致的地方并给出修改建议。

[粘贴待检查的代码]
```

### 设计决策咨询

```
在当前设计约束下，[描述设计问题]，有哪些可行方案？
对每个方案给出视觉效果描述和与现有风格的兼容性评估。
不需要直接给出代码，先给方案供选择。
```

## 设计图转 AI 指导

已有设计稿（Figma / 截图 / 草图）时，不要直接扔给 AI 让它"照着写代码"——这样输出质量极不稳定。正确做法是先把设计图翻译成结构化描述，再交给 AI 执行。

### 第一步：从设计图提取信息

对着设计稿，按以下维度逐项记录：

```
【布局结构】
- 整体分几个区域？各区域的宽高比或比例关系？
- 是 Flex 还是 Grid？主轴方向？
- 有无固定宽度列或自适应列？

【组件清单】
- 列出页面中所有出现的组件（按钮 / 输入框 / 表格 / 卡片…）
- 每个组件的变体：尺寸、状态、内容类型

【间距与对齐】
- 各区域的内外边距大概是多少（估算到基准单位的倍数）？
- 元素之间的对齐方式（左对齐 / 居中 / 两端对齐）？

【视觉细节】
- 出现了哪些颜色？与风格指南是否一致？
- 圆角、阴影、边框有没有特殊处理？
- 图标、图片的尺寸和位置规律？

【交互说明】
- 设计图中暗示了哪些交互（hover 样式、折叠、弹窗）？
```

### 第二步：生成提示

把提取的信息组织成给 AI 的指令：

```
我有一份设计稿，以下是结构描述：

【布局】[填写第一步提取的布局结构]
【组件】[填写组件清单]
【视觉细节】[填写颜色、间距、圆角等]
【交互】[填写交互说明]

请按此描述生成代码，同时遵守以下项目约束：
[附上基础上下文模板内容]

如果描述中有模糊的地方，先列出你的理解和假设，再给出代码。
```

### 第三步：对比校验

AI 生成后，将输出与设计图对比，重点检查：

- 布局比例是否接近（不需要像素级精确）
- 颜色是否来自项目约束，没有引入新色值
- 组件状态是否完整（不只是默认态）
- 间距节奏是否符合基准单位

## 从零创建设计稿指导

没有设计稿、需要 AI 从无到有产出设计方案时，关键是**分阶段推进**，每个阶段对齐一次，不要一步要求出完整页面。

### 阶段一：确定信息架构

先让 AI 梳理页面要呈现的内容，不涉及任何视觉细节：

```
我需要设计一个 [页面名称]，用途是 [一句话描述]。

目标用户：[用户类型]
核心操作：[用户在这个页面要完成什么]
需要展示的信息：[数据、状态、操作入口等]

请列出这个页面的信息层级和主要区域划分，不需要涉及样式。
```

确认信息架构后，再进入下一阶段。

### 阶段二：确定布局结构

在信息架构基础上，让 AI 给出布局方案：

```
基于以上信息架构，给出 2–3 种布局方案，每种方案描述：
- 整体结构（几栏、导航位置、内容区划分）
- 适合的场景或用户行为模式
- 与项目现有页面的一致性

用文字或 ASCII 示意图描述，不需要代码。
```

选定布局后，进入下一阶段。

### 阶段三：生成高保真代码

布局和内容确认后，结合项目约束生成代码：

```
基于以上确认的布局方案，按项目设计约束生成完整页面代码。

[附上基础上下文模板]

要求：
- 使用真实的占位内容（不要写 "Lorem ipsum"，用符合场景的示例数据）
- 所有组件状态完整（默认 / 空态 / 加载态）
- 响应式断点：[如 768px / 1280px]
```

### 阶段四：迭代细化

整体结构确认后，逐个组件或区域细化：

```
上面的页面整体方向对了，现在细化 [具体区域]：
- 当前问题：[描述哪里不对]
- 期望效果：[具体说明，不要说"更好看"]
- 约束：不改变其他区域
```

**原则**：每次迭代只改一处，问题定位清晰，AI 不会连带破坏其他部分。

---

## 使用建议

- 约束越具体，输出越稳定——模糊的描述会让 AI 自行发挥
- 提供现有组件代码作为参考，比纯文字描述效果好得多
- 如果 AI 输出偏离风格，优先检查约束中是否有遗漏，而不是反复纠正
- 同一个项目的提示上下文固定下来后，存入此文件复用，不要每次重新写
