# AI 辅助 UI 设计

AI 不会自动记住项目的设计风格。每次对话从零开始，它对你的色板、字体规范、圆角决策一无所知——除非你在提示中显式提供。

理解这一点，是用好 AI 做 UI 的前提。

---

## AI 如何"学习"设计风格

**训练阶段（模型内置）**：AI 通过大规模设计数据集习得视觉模式，能识别颜色关系、排版节奏、组件结构，能做风格迁移。这是它的通用能力。

**使用阶段（上下文学习）**：模型不会为你重新训练。它通过提示中给出的示例推断风格规律（few-shot）。你提供的上下文越具体，输出越符合项目风格。

**结论**：AI 的"风格记忆"= 你在每次提示中提供的设计信息。没有上下文，它就用训练数据里的"平均风格"填充。

---

## 如何向 AI 传递设计风格

| 方法 | 效果 |
|------|------|
| 提供设计令牌（色值、字体、间距） | 最稳定，AI 有明确的数值依据 |
| 提供现有组件代码 | AI 从代码推断整体设计语言，比描述更准确 |
| 风格描述词 | 快速定调，适合早期探索 |
| 负面约束 | 明确排除不想要的风格，减少纠偏次数 |
| 截图或参考图 | 辅助用，AI 解读图片不如结构化描述稳定 |

最有效的组合：**设计令牌 + 一个现有组件示例 + 负面约束**。

## 设计资产如何组织

把设计规范存进仓库，AI 工作前读取作为上下文。本目录的组织方式：

```
02-design/
├── ui-style-guide.md       风格基准——色彩、字体、间距、圆角等核心维度
├── design-decisions.md     决策日志——记录为什么这样设计，防止被无意推翻
└── ui-design-with-ai.md    本文件——原理与方法概述

04-prompt/design/
└── ai-design-prompt.md     提示模板——生成 UI、转译设计图、从零创建的具体提示
```

**使用顺序**：先在 `ui-style-guide.md` 定义风格基准 → 用 `ai-design-prompt.md` 的模板生成 UI → 重要决策写进 `design-decisions.md`。

---

## 原则

AI 生成的 UI 质量上限，等于你给它的设计上下文的质量。规范不写进文件，每次生成都是在猜。
