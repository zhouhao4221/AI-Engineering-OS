# AI 辅助 UI 设计风格管理

AI 不会自动记住项目的设计风格。本文档记录如何让 AI 学习并持续保持 UI 设计一致性，以及如何在项目中管理设计决策。

---

## AI 如何学习 UI 设计风格

### 训练阶段（模型内置能力）

AI 通过大规模设计数据集学习视觉模式：
- 颜色、排版、间距、组件结构的统计规律
- 图文对（设计截图 + 描述）建立语义与视觉的映射
- 风格迁移：从已有设计提取色调、圆角、阴影等特征向量

### 使用阶段（上下文学习）

模型不会重新训练，而是通过提示中的示例推断风格规律（few-shot）。这意味着：

**AI 的"记忆"完全依赖你在上下文中提供的信息。**

---

## 如何引导 AI 保持项目风格

| 方法 | 做法 |
|------|------|
| 提供设计令牌 | 传入色值、字体规范、间距系统（Token 文件） |
| 截图或参考图 | 上传现有 UI，让 AI 分析并提取风格规律 |
| 风格描述词 | "极简主义、高对比度、无圆角、系统字体" |
| 组件示例 | 提供一个 Button/Card 组件，AI 推断整体设计语言 |
| 负面约束 | 明确说"不要渐变、不要圆角超过 4px、不要阴影" |

## 项目设计资产结构

将设计规范存入仓库，AI 每次工作前读取作为上下文：

```

├── ui-design-with-ai.md     AI 辅助设计流程与管理方法         ← 本文件

```

## 设计决策管理

### 问题：AI 的"设计决策失忆"

每次对话重新开始，AI 不知道"为什么当时这样设计"。解决方案是将决策文档化并存入仓库。

### 设计决策记录格式（ADR 变体）

```markdown
## 决策：[组件/属性名称]
日期：YYYY-MM
背景：面临的设计问题是什么
选项：考虑了哪些方案
决策：最终选择了什么
原因：为什么选这个（风格定位、用户群体、技术约束）
AI 参与：AI 在这次决策中承担了什么角色
```

### 示例

```markdown
## 决策：圆角尺寸
日期：2026-01
背景：统一全局圆角，避免组件风格不一致
选项：4px（克制）/ 8px（友好）/ 12px（圆润）
决策：4px
原因：产品定位偏向专业工具，8px 以上视觉过于柔和，与品牌调性不符
AI 参与：AI 生成了三个方案的视觉对比描述，最终由设计师决策
```

## AI 设计提示模板结构

生成 UI 时，将以下信息作为提示前置上下文：

```
【设计约束】
- 色板：主色 #xxx，背景 #xxx，文字 #xxx
- 字体：系统字体栈，正文 14px，标题 20px/24px
- 间距：基准单位 4px，组件内边距 12px/16px
- 圆角：统一 4px，按钮除外（0px）
- 阴影：不使用，用边框代替层级关系
- 图标：仅用 [图标库名称]，不混用风格

【禁止项】
- 不使用渐变背景
- 不使用大面积彩色区块
- 不在数据展示组件上使用动画

【参考组件】
[粘贴一个现有组件代码或截图描述]
```

## 一致性检查清单

在 AI 生成 UI 后，用以下问题验证输出：

- [ ] 颜色是否来自设计令牌，没有硬编码值？
- [ ] 字体大小是否符合规范的尺寸阶梯？
- [ ] 间距是否是 4px 基准的倍数？
- [ ] 圆角、阴影是否与设计决策一致？
- [ ] 组件结构是否可复用，还是一次性写死？

---

## 原则

AI 生成的 UI 质量上限 = 你给它的设计上下文的质量。规范不写进仓库，每次生成都是在猜。
